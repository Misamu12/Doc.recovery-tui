RecoveryTUI — Backup & Restore Linux

Sauvegardez et restaurez vos applications Linux installées en quelques secondes. Interface TUI intuitive, CLI puissant, et support de 5 gestionnaires de paquets.

✓ APT / Snap ✓ Flatpak ✓ Pacman ✓ DNF ✓ GitHub Storage

Installation #

Deux méthodes d'installation : script automatique ou manuelle avec virtualenv.

Script automatique

bash 
curl -sSL https://github.com/Misamu12/recovery-tui/install.sh | bash

Installation manuelle (recommandée)

1

Extraire et naviguer dans le dossier

bash
unzip recovery_tui_v1.0.zip && cd recovery_tui
2

Créer un environnement virtuel

bash
python3 -m venv .venv
source .venv/bin/activate
3

Installer les dépendances

bash
pip install -r requirements.txt
pip install -e .
4

Vérifier l'installation

bash
recovery-tui --version
# recovery-tui, version 1.0.0
⚠ Sur Ubuntu 23+ ou Debian 12+, utilisez pip install --break-system-packages si vous n'utilisez pas de virtualenv.

backup #

Scanne les applications installées et génère un fichier snapshot JSON (ou .json.gz).

CMD recovery-tui backup [OPTIONS]
OptionTypeRequisDescription
--label, -lstringoptionnelNom du snapshot (ex: "avant-migration"). Timestamp auto si absent.
--output, -opathoptionnelDossier de destination. Défaut: ~/.local/share/recovery-tui/snapshots/
--compressflagoptionnelCompresse le snapshot en .json.gz.
--githubflagoptionnelPousse aussi le snapshot vers GitHub après sauvegarde locale.
--managers, -mstringoptionnelGestionnaires à scanner, séparés par virgule. Ex: apt,snap

Exemples

bash
# Backup simple
recovery-tui backup

# Avec label et compression
recovery-tui backup --label "ubuntu-22.04" --compress

# APT et Snap uniquement, push GitHub
recovery-tui backup --managers apt,snap --github

# Vers un dossier personnalisé
recovery-tui backup --output /mnt/backup/

Fichier de sortie

restore #

Installe toutes les applications d'un snapshot sur le système courant.

CMD recovery-tui restore [SNAPSHOT] [OPTIONS]
OptionTypeRequisDescription
SNAPSHOTpathoptionnelChemin vers un fichier .json ou .json.gz. Si absent, utilise le plus récent.
--snapshot-label, -sstringoptionnelRestaurer par label partiel.
--dry-runflagoptionnelSimule la restauration sans installer aucun paquet.
--forceflagoptionnelBypass la vérification d'espace disque.
--skip-confirmflagoptionnelSupprime la confirmation interactive (utile en CI/CD).

Exemples

bash
# Restaurer le dernier snapshot
recovery-tui restore

# Dry-run pour prévisualiser
recovery-tui restore --dry-run

# Depuis un fichier précis
recovery-tui restore ./snapshot_ubuntu-22.04.json

# Par label, sans confirmation (CI/CD)
recovery-tui restore -s "avant-migration" --skip-confirm

# Forcer malgré espace insuffisant
recovery-tui restore --force --skip-confirm
✓ La commande restore vérifie automatiquement l'espace disque disponible vs requis avant d'installer quoi que ce soit. Utilisez --dry-run pour tester sans risque.

Scanner — core/scanner.py #

Le scanner détecte automatiquement les gestionnaires disponibles et les scanne en parallèle via asyncio.gather().

Structure AppEntry

python
AppEntry = {
  "name":        str,   # "vim"
  "version":     str,   # "9.0.1749-1"
  "manager":     str,   # "apt" | "snap" | "flatpak" | "pacman" | "dnf"
  "size_mb":     float, # 25.0
  "category":    str,   # "Dev" | "Essential" | "Multimedia" | ...
  "install_cmd": str,   # "sudo apt install -y vim"
  "config_files": list  # ["~/.vimrc"]
}

API Python

python
import asyncio
from recovery_tui.core.scanner import scan_all

# Scan tous les gestionnaires disponibles
apps = asyncio.run(scan_all())

# Scan uniquement APT et Snap
apps = asyncio.run(scan_all(managers=["apt", "snap"]))

# → liste de AppEntry dicts, triée par (manager, name)

Snapshots — core/backup.py #

python — API complète
from recovery_tui.core.backup import build_snapshot, save_snapshot, load_snapshot
from recovery_tui.storage.local import save_local, list_local_snapshots

# Créer et sauvegarder
snapshot = build_snapshot(apps, label="test", compress=False)
path = save_local(apps, label="test")

# Charger
data = load_snapshot(path)
apps = data["apps"]

# Lister (tri newest-first)
snaps = list_local_snapshots()  # → [{"label","created_at","path",...}]

GitHub Storage #

Poussez vos snapshots vers un repository GitHub privé pour les restaurer sur n'importe quel système.

Configuration

bash
# 1. Créer un token GitHub (Settings → Developer → Personal access tokens)
#    Permissions requises : repo (read/write)

# 2. Configurer RecoveryTUI
recovery-tui config set github.token ghp_xxxxxxxxxxxxxxxx
recovery-tui config set github.repo monuser/recovery-backup
recovery-tui config set github.branch main

Utilisation

bash
# Push lors du backup
recovery-tui backup --github

# Pull depuis le TUI : écran Restore → bouton "☁ From GitHub"
⚠ Ne committez jamais votre token GitHub dans le code source. Il est stocké uniquement dans ~/.config/recovery-tui/config.yaml qui est ignoré par git.

config.yaml — Référence complète #

Fichier généré automatiquement à ~/.config/recovery-tui/config.yaml

yaml
storage:
  type: local                  # "local" | "github"
  local_path: ~/.local/share/recovery-tui/snapshots

github:
  token: ""                    # ghp_xxxxxxxxxx
  repo:  ""                    # username/repo-name
  branch: main

scanner:
  enabled_managers:
    - apt
    - snap
    - flatpak
    - pacman
    - dnf
  exclude_packages: []        # ex: ["linux-firmware"]

ui:
  theme: dark
  language: en

Erreurs courantes #

ERRModuleNotFoundError: No module named 'recovery_tui'

Le package n'est pas installé dans le bon virtualenv. Vérifiez que source .venv/bin/activate est actif puis relancez pip install -e . depuis le dossier contenant setup.py.

ERRInvalid CSS property 'variant'

variant est un attribut Python, pas une propriété CSS Textual. Supprimez les blocs #btn-X { variant: ... } du CSS inline dans les screens.

ERRpermission denied while trying to connect to Docker

Votre utilisateur n'est pas dans le groupe docker. Exécutez sudo usermod -aG docker $USER && newgrp docker, puis reconnectez-vous.

ERRGitHub token not configured / ValueError

Configurez le token via recovery-tui config set github.token ghp_xxx et le repo via recovery-tui config set github.repo user/repo.
